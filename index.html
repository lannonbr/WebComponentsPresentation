Components of HTML

Components have existed since the beginning of the web. These include div, img, p, etc.

but what if I want my own?

Enter Web Components.

What we've had in the past

Following, we would need to use something like JQuery to add some buttons and make it act like a carousel

What we have now

Now we have semantic meaning to elements and we can use native APIs to be able to interact with these with just plain Javascript

Quick aside

Since we have just one DOM (Document Object Model), everything is encompassed in a single entity. This can become a problem with CSS that can bleed into elements you don't want to. Good thing is we have a way to fix that with Web Components.

The Shadow-DOM

The styles inside the template tags are shut off from the rest of the document. nothing gets in or out.

Browser Support

Custom Elements v1 and Shadow Dom v1 are in the newest stable Chrome release (54). the original specs are in Chrome, Firefox (with a flag), and Safari. Luckily, all modern browsers are planning on implementing the v1 specs

For backwards compatibility, look into these polyfills.
